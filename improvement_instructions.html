USBR Review App - Improvement Instructions

Comprehensive guide to enhancing the app.py Streamlit application

Overview

This document provides detailed recommendations for improving the app.py file in the USBR Review project. The application currently displays environmental monitoring data from various sites using Streamlit, matplotlib, and plotly visualizations.

1. Code Duplication and Refactoring Opportunities

HIGH Refactor the get_db function

The current implementation uses repetitive if/else statements that can be simplified with a dictionary mapping approach:

# Current approach (repetitive) def get_db(all_site_data:DB_Reader, site_name) -> pd.DataFrame: daqm_data = pd.DataFrame() start_time = process_time() if site_name == 'Baggs': daqm_data = all_site_data.baggs if site_name == 'Boulder': daqm_data = all_site_data.boulder # ... repeated for each site # Improved approach (dictionary mapping) SITE_DATA_MAPPING = { 'Baggs': 'baggs', 'Boulder': 'boulder', 'Cora': 'cora', # ... etc } def get_db(all_site_data: DB_Reader, site_name) -> pd.DataFrame: start_time = process_time() site_attr = SITE_DATA_MAPPING.get(site_name) if site_attr: daqm_data = getattr(all_site_data, site_attr) else: daqm_data = pd.DataFrame() end_time = process_time() print(f'{site_name} read time: {(end_time-start_time):.8f} seconds.') return daqm_data

HIGH Eliminate duplicate functions

The clean_column and nan_check functions exist in both app.py and PlotGenerator.py. Remove them from app.py and use the ones in PlotGenerator.py through the instance.

2. Code Organization and Maintainability

MEDIUM Split the large app.py file

The file is quite large with many functions. Consider organizing it into modules:

  • data_processing.py - Data handling and cleaning functions
  • visualization.py - Plotting functions
  • ui_components.py - Streamlit UI components
  • utils.py - Utility functions

MEDIUM Manage global variables

Replace global variables with configuration parameters or class attributes:

class AppConfig: DAYS_LIMIT = 7 TICK = 3 TIME_ZONE = pytz.timezone("US/Mountain") PLOT_SIZE = (18, 4)

LOW Make paths configurable

Replace hardcoded paths with configurable ones:

# Instead of hardcoded paths like: image_self = Image.open(r"Data/SatelliteImage/self.jpg") # Use: from pathlib import Path IMAGE_PATH = Path("Data/SatelliteImage") image_self = Image.open(IMAGE_PATH / "self.jpg")

4. Error Handling and Logging

HIGH Implement proper logging

Replace print statements with proper logging:

import logging logger = logging.getLogger(__name__) # Instead of: print(f'{site_name} read time: {(end_time-start_time):.8f} seconds.') # Use: logger.info(f'{site_name} read time: {(end_time-start_time):.8f} seconds.')

MEDIUM Add comprehensive error handling

Wrap critical operations in try/except blocks:

try: merged_df = get_db(all_site_data, str(selected_site)) if merged_df.empty: st.warning(f"No data found in the database of {selected_site} site.") return except Exception as e: logger.error(f"Error loading data for {selected_site}: {e}") st.error(f"Failed to load data for {selected_site}. Please check the logs.") return
